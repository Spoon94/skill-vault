### 3. 以 "-ing" 类副词短语收尾的肤浅分析（中文里是"…着 / …地"结构）

需注意：*突出/强调/彰显着…、确保…、反映/象征着…、为…做出贡献、培养/促进着…、
涵盖着…、展示着…* 一句话末尾再加一个分词式状语装深沉，是 AI 翻译腔。

改写前：寺庙的蓝色、绿色和金色色调与该地区的自然美景产生共鸣，象征着德克萨斯
州的蓝帽花、墨西哥湾和多样化的德克萨斯州景观，反映了社区与土地的深厚联系。
改写后：寺庙使用蓝、绿、金三色。建筑师说这是为了呼应当地的蓝帽花和墨西哥湾
海岸。

### 4. 宣传与广告式语言

需注意：*拥有（夸张用法）、充满活力的、丰富的（比喻）、深刻的、增强、展示、
体现、致力于、自然之美、坐落于、位于…的中心、开创性的（比喻）、著名的、令人
叹为观止的、必游之地、迷人的、底蕴深厚。*

改写前：坐落在埃塞俄比亚贡德尔地区令人叹为观止的区域内，Alamata Raya Kobo

改写后：Alamata Raya Kobo 是埃塞俄比亚贡德尔地区的一座城镇，以每周集市和 18
世纪的教堂闻名。

### 5. 模糊归因与含糊措辞

需注意：*行业报告显示、观察者指出、专家认为、一些批评者认为、多个来源/出版物、
有研究表明（不给具体研究）。*

改写前：由于其独特特征，浩来河引起了研究人员和保护主义者的兴趣。专家认为它

改写后：根据中国科学院 2019 年的调查，浩来河中有多种特有鱼类。

### 6. 公式化的"挑战与未来展望"段落

需注意：*尽管其…面临若干挑战…、尽管存在这些挑战、挑战与遗产、未来展望、
任重而道远。*

改写前：尽管工业繁荣，Korattur 面临着城市地区典型的挑战，包括交通拥堵和水

改写后：2015 年三个 IT 园区开业后，交通拥堵加剧。市政公司于 2022 年启动了
雨水排水项目。

## 语言语法类

### 7. 高频 AI 词汇

会成团出现：*此外、与…保持一致、至关重要、深入探讨、强调、持久的、增强、
培养、获得、突出（动词）、相互作用、复杂/复杂性、关键（形容词）、格局（抽象
名词）、关键性的、展示、织锦（抽象名词）、证明、强调（动词）、宝贵的、充满
活力的、赋能、助力、打造、护航、布局、生态、闭环、底层逻辑、抓手、范式、
切入点。* 单独出现没事，一段里挤了三四个就是 AI 自白。

### 8. 系动词回避

需注意：*作为/代表/标志着/充当 [一个]、拥有/设有/提供 [一个]。* 用复杂结构
替代"是/有"。

改写前：Gallery 825 作为 LAAA 的当代艺术展览空间。画廊设有四个独立空间，
拥有超过 3000 平方英尺。
改写后：Gallery 825 是 LAAA 的当代艺术展览空间，有四个房间，共 3000 平方
英尺。

### 9. 否定式排比

"不仅…而且…"、"这不仅仅是…，而是…"、"与其说是…不如说是…" 滥用。

改写前：这不仅仅是节拍在人声下流动；它是攻击性和氛围的一部分。这不仅仅是
一首歌，而是一种声明。
改写后：沉重的节拍加强了攻击性。

### 10. 三段式法则过度

AI 习惯凑三件：*创新、灵感与洞察 / 速度、规模与质量 / 高效、智能、可靠。*
试着用两项或四项。

改写前：活动包括主题演讲、小组讨论和社交机会。与会者可以期待创新、灵感和
行业洞察。
改写后：活动有演讲和小组讨论，会议之间留出非正式交流的时间。

### 11. 同义词循环

AI 怕重复，于是"主人公—主角—中心人物—英雄"循环换称。读者只会被绕晕。

改写前：主人公面临许多挑战。主要角色必须克服障碍。中心人物最终获得胜利。
英雄回到家中。
改写后：主人公面临许多挑战，但最终获胜并回到家中。

### 12. 虚假范围

"从 X 到 Y" 但 X 和 Y 根本不在一个尺度上。

改写前：我们穿越宇宙的旅程将我们从大爆炸的奇点带到宏伟的宇宙网，从恒星的诞生
和死亡到暗物质的神秘舞蹈。
改写后：这本书涵盖大爆炸、恒星形成和当前的暗物质理论。

### 13. 被动语态和无主语句

中文相对少见"被"字句，AI 却经常用："系统会被优化"、"配置文件无需被创建"。
改成主动语态。

改写前：无需配置文件。结果会被自动保留。
改写后：你不需要配置文件，系统会自动保留结果。

## 风格格式类

### 14. 破折号过度

AI 喜欢用 `—` 或 `——` 模仿"有力的销售文案"。**最终稿应当不含 `—` 和 `–`。**
依次替换为：句号（新起一句）、逗号（紧凑插入）、冒号（解释引出）、括号（真
正的旁注）。

改写前：这个术语主要由荷兰机构推广——而不是由人民自己。你不会说"荷兰，欧洲"
作为地址——但这种错误标记仍在继续——即使在官方文件中。
改写后：这个术语主要由荷兰机构推广，而不是由人民自己。你不会把"荷兰，欧洲"
当地址写，但这种错误标记在官方文件中仍然存在。

### 15. 粗体滥用

每个关键词都加粗，等于没加粗。

改写前：它融合了 **OKR（目标和关键结果）**、**KPI（关键绩效指标）** 和视觉
战略工具，如 **商业模式画布（BMC）** 和 **平衡计分卡（BSC）**。
改写后：它融合了 OKR、KPI 和视觉战略工具，如商业模式画布和平衡计分卡。

### 16. 内联标题式垂直列表

`- **要点：** 一句重复要点的话` 这种排版。

改写前：
- **用户体验：** 用户体验通过新界面得到显著改善。
- **性能：** 性能通过优化算法得到增强。
- **安全性：** 安全性通过端到端加密得到加强。

改写后：更新改进了界面、通过优化算法加快了加载速度，并加入了端到端加密。

### 17. 装饰性 emoji

🚀 / 💡 / ✅ 用在标题或项目符号前，没有任何信息增益。删掉。

### 18. 标点不一致

中文段落里混入英文逗号 `,`、英文句号 `.`、英文引号 `""`。AI 在中英文混排时
最容易翻车。统一标点：

- 中文句子用 `，。；：？！` 和 `「」/""`
- 中英混排时，**英文术语前后的标点用中文标点**（"我用的是 Python，因为…"
  而不是 "我用的是 Python, 因为…"）

## 交流痕迹类

### 19. 聊天机器人的对话残留

需注意：*希望这对您有帮助、当然！、一定！、您说得完全正确！、您想要…、请告诉
我、这是一个…、如果您有任何问题、随时为您服务。* 这是聊天框里的话被粘贴到正
文里。

改写前：这是法国大革命的概述。希望这对您有帮助！如果您想让我扩展任何部分，
请告诉我。
改写后：法国大革命始于 1789 年，财政危机和粮食短缺导致广泛动荡。

### 20. 知识截止日期免责声明 + 推测填空

需注意：*截至 [日期]、根据我最后的训练更新、虽然具体细节有限/稀缺…、基于可
用信息…、其早年经历未被广泛记录、推测可能是在…、应该是在…左右。*

两个相关痕迹：(a) 训练截止日期免责声明残留。(b) 找不到来源时写"找不到来源"
这件事本身，然后编一段听起来合理的填空。

改写前：虽然关于公司成立的具体细节在现成资料中没有广泛记录，但它似乎是在
20 世纪 90 年代的某个时候成立的。
改写后：根据注册文件，该公司成立于 1994 年。

### 21. 谄媚 / 卑躬屈膝的语气

改写前：好问题！您说得完全正确，这是一个复杂的话题。关于经济因素，这是一个
很好的观点。
改写后：您提到的经济因素在这里是相关的。

## 填充与修辞类

### 22. 填充短语

| 改写前 | 改写后 |
|---|---|
| 为了实现这一目标 | 为了实现这一点 / 为此 |
| 由于…的事实 / 由于…的原因 | 因为… |
| 在这个时间点 | 现在 |
| 在您需要帮助的情况下 | 如果您需要帮助 |
| 系统具有处理…的能力 | 系统可以处理… |
| 值得注意的是数据显示 | 数据显示 |
| 不难看出 / 显而易见 | （直接说结论） |
| 在某种程度上 / 在一定程度上 | （多半可以删） |

### 23. 过度限定

改写前：可以潜在地可能被认为该政策可能会对结果产生一些影响。
改写后：该政策可能会影响结果。

改写前：公司的未来看起来光明。激动人心的时代即将到来，他们继续追求卓越的旅
程。这代表了向正确方向迈出的重要一步。
改写后：该公司计划明年再开设两个地点。

### 25. 修辞性权威话术

需注意：*真正的问题是、归根结底、本质上、究其根本、归根到底、说到底、关键

通观点穿了大衣。

改写前：真正的问题是团队能否适应。归根结底，组织准备程度才是关键。

需注意：*让我们深入探讨、让我们一起看看、让我们来分析、接下来、闲话少说、

## 中文独有的 AI 痕迹

### 29. 机械动词大全

中文 LLM 写作的"代号词汇"。看到就要警觉，想想能不能换成更具体的动作：

| 机械动词 | 替换方向 |
|---|---|
| 打造 / 倾力打造 | 做、做出、建（具体动作） |
| 赋能 | 给…加上某项能力（说明是什么能力） |
| 助力 | 帮助 / 让… |
| 护航 | 支持 / 保证 |
| 深入探讨 / 深入分析 | 讨论 / 分析 |
| 不断演变 / 日新月异 | （多半可删） |
| 持续发力 / 持续深耕 | 继续做 |
| 全面布局 | 在…做 |
| 重塑 / 颠覆 | 改变 / 换掉 |
| 引领 / 领跑 | 比别人早 / 在前面 |
| 拥抱 + 抽象名词 | （删掉"拥抱"） |

### 30. 的的不休

连续三个及以上"的"叠加，是 AI 译腔特征：
> 一个**充满活力的**、**多元化的**、**开放的**社区

改成口语化的并列即可：
> 一个有活力、多元、开放的社区

类似的还有连续"地"（"快速地、准确地、高效地完成"）和连续"了"
（"完成了、提交了、验证了"）。

### 31. 翻译腔结构

- **悬空的"作为…"开头**：
  *作为一名资深开发者，这个工具非常好用。* → *我作为资深开发者用过，这工具
  很好用。* / *资深开发者会觉得这工具好用。*
- **滥用"被"字句**：*这个问题被解决了* → *这个问题解决了 / 我把这个问题解
  决了*。
- **"是…的"判断句堆叠**：*这个决定是基于数据的、是经过深思熟虑的、是符合
  长期利益的。* → *这个决定有数据支撑，也是认真权衡过的，对长期是有利的。*
- **超长定语**：*一个能够帮助开发者快速构建可扩展、可维护、易于部署的现代
  化 Web 应用的框架* → *一个让开发者快速做出现代化 Web 应用的框架，方便扩
  展、维护和部署。*

### 32. 括号注释 AI 化

英文术语后都跟（中文翻译），过密。规则：术语第一次出现时给一次注释就够，后
面直接用。如果原文是技术文档面向开发者，往往连第一次都不需要。

改写前：使用 GitHub（一个代码托管平台）的 Actions（工作流自动化工具）来
跑 CI（持续集成）。
改写后：用 GitHub Actions 跑 CI。

### 33. AI 风格的中文标点

- 全角引号 `""` 包英文：`他说"hello"` → `他说 "hello"`（英文外层用半角并
  在中文两侧留空格）
- 中文逗号顿号混乱：列举短词用 `、`，列举短句用 `，`
- 多余的空格：中文之间不要加空格（"我 觉 得" 是 AI 残留）
- 中英之间留半角空格是合规习惯，不算 AI 痕迹

## 不要误判的情况（假阳性）

人类作者也会自然落入上面某些模式。下列情况单独出现 **不构成** AI 证据：

- 行文流畅、语法规范 —— 有编辑能力的写作者本来就这样
- 单独一个破折号、单独一个"然而"、单独一个长定语
- 偏书面、偏学术的词汇（"然则、诚然、譬如、姑且不论"）—— AI 用的是 *特定*
  套话，不是所有书面词
- 单独一个短句强调 —— 只在连续多个短句堆砌时才算痕迹

**人类痕迹**（看到这些要保留，不要"优化"掉）：

- 具体、不寻常、难以编造的细节
- 矛盾的、未解决的情绪
- 时代相关的俚语、梗、圈内黑话
- 真诚的离题、括号里的自我吐槽、自我修正
- 句长差异大、节奏不齐
- 第一人称表达观点（"我一直觉得…"、"我搞不懂为什么…"）
